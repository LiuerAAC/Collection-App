# v0.4 更新目标：Obsidian 数据端口与 MARD 拼豆库

日期：2026-06-13

## 1. 本轮范围

本轮先完成文档层设计，不改代码。

下一次实现时，目标是把 Obsidian vault 中已经先行存储的数据接入 Web Dataset，并新增拼豆库存管理能力。

## 2. 背景

当前 Web 端已经具备 PWA、本地存储、Supabase 登录/同步和图片上传方向，但网络端长期稳定性仍需继续验证。为了避免数据因为网络端迭代而不稳定，当前真实数据先存放在 Obsidian vault：

```text
/Users/yuxuancao/Library/Mobile Documents/iCloud~md~obsidian/Documents/Life
```

Web App 下一步不应直接替代 vault，而应先读取、解析、匹配和上传 vault 中的结构化数据。

## 3. Obsidian 数据端口

### 3.1 目标

新增一个导入/同步端口，从 vault 获取全部相关数据，并上传到 Web Dataset。

### 3.2 数据来源

优先读取：

1. `10_Collections/Items/**`：已入库藏品。
2. `10_Collections/Pending/**`：待入库藏品草稿。
3. `10_Collections/Orders/**`：订单。
4. `10_Collections/Sources/**`：图纸来源。
5. `20_Attachments/**`：藏品图片、图纸图片、附件。
6. `.base` 文件：Base 视图配置只作为辅助，不作为唯一数据源。

### 3.3 解析内容

每个 Markdown 文件需要解析：

1. YAML frontmatter / Properties。
2. Obsidian 内链，例如 `[[Card 0001]]`。
3. 图片引用，例如 `![[image.jpg]]` 或 Markdown 图片。
4. 文件路径所属分类，例如 `Items/Cards`、`Pending/Perler`。
5. Dataview/Base 可读字段。

### 3.4 匹配规则

导入时需要建立稳定匹配，避免重复上传：

1. 优先使用文件路径和文件名作为 `vaultPath`。
2. 使用 Obsidian 文件名作为默认展示名称。
3. 订单优先按 `order_no` 匹配；没有订单号时使用 `vaultPath`。
4. 藏品优先按 `vaultPath` 匹配；后续可补充 `externalId`。
5. 图片通过附件相对路径和文件 hash 匹配。
6. 已上传过的数据再次导入时做更新，不生成副本。

### 3.5 上传策略

建议流程：

1. 读取 vault。
2. 生成本地解析结果。
3. 展示导入预览，包括新增、更新、跳过、冲突。
4. 用户确认后上传 Dataset。
5. 图片资产单独上传到 Storage。
6. 写入导入日志，记录时间、数量、失败项和来源路径。

### 3.6 冲突策略

第一版采用保守策略：

1. Web 端已有数据但 vault 更新时间更晚时，标记为可更新。
2. Web 端已有数据且用户手动修改过时，进入冲突列表。
3. 失败项不阻塞整批导入。
4. 不自动删除 Web 端数据，除非未来支持显式删除同步。

## 4. Dataset：MARD 221 色拼豆库

### 4.1 目标

在网页 Dataset 中新增 MARD 221 色拼豆库，用于记录色号、颜色、库存、消耗和统计排行。

### 4.2 色号范围

色号库包含 `221` 个 MARD 色号。

筛选分组：

1. 全部
2. A
3. B
4. C
5. D
6. E
7. F
8. G
9. H
10. M

### 4.3 色号字段

每个色号建议字段：

1. 色号，例如 `A001`。
2. 分组，例如 `A`。
3. 颜色名称。
4. HEX 色值。
5. RGB 色值。
6. 纯色色卡图片或可渲染色块。
7. 当前库存。
8. 累计入库。
9. 累计消耗。
10. 最近更新时间。

### 4.4 库存交互

每个豆库条目有一个加号按钮：

1. 点击一次增加 `1000` 颗库存。
2. 增加库存时写入库存日志。
3. 日志记录色号、数量、操作类型、时间、来源。
4. 后续可扩展手动减少、盘点校正和备注。

### 4.5 图纸消耗扣减

拼豆图纸识别后，需要提取每个色号的用量。

扣减流程：

1. 用户上传或选择拼豆图纸。
2. 系统识别图纸中的色号和数量。
3. 生成识别结果预览。
4. 用户确认后创建消耗日志。
5. 从对应色号库存中扣减。
6. 消耗日志关联到对应拼豆藏品。

第一版必须允许人工确认和手动修正，不应直接静默扣库存。

### 4.6 统计新增

分析页新增色号使用排行：

1. 按累计消耗排行。
2. 按最近项目消耗排行。
3. 按库存不足预警排序。
4. 支持按色号组筛选。
5. 可从排行跳转到色号详情或相关拼豆藏品。
