# Collection App v0.1 开发文档

日期：2026-06-09  
形态：Web App / PWA  
目标：先用 PWA 完成私人收藏管理核心流程，支持本地优先、离线浏览、后续在线同步预留

## 1. 当前技术架构

1. 前端：React + TypeScript + Vite
2. PWA：`manifest.webmanifest` + `sw.js`
3. 本地存储：`localStorage`
4. 状态管理：React Context
5. 构建输出：`dist/`

## 2. 当前页面结构

### 2.1 顶部导航

1. `Collection App`
2. 仓库
3. Gallery
4. 分析
5. 设置

说明：

1. 顶部导航固定在页面上方。
2. 四个主入口使用圆形图标。
3. 桌面和手机端共用同一套导航结构。

### 2.2 仓库

仓库页已拆成三个子页面：

1. 默认仓库
2. 新增藏品
3. 订单管理

#### 默认仓库

1. 默认显示全部藏品。
2. 支持按分类筛选。
3. 支持按标签筛选。
4. 每个藏品卡片默认只显示图片。
5. 点击藏品进入详情弹层。

#### 藏品详情

1. 展示图片。
2. 展示状态、卡价、购买时间、购买金额、模板字段等信息。
3. 支持修改。
4. 支持删除。
5. 修改图片支持从图库选择或拍照上传。

#### 新增藏品

1. 先选择类别。
2. 默认重点支持卡牌流程。
3. 图片与文本信息改为上下排列。
4. 上传图片统一缩放到固定展示尺寸。
5. 卡牌类字段包括：
   - 卡牌公司
   - 系列
   - 年份
   - 球员名称
   - 队伍
   - 卡牌编号
   - 折射类型
   - 特卡名称
   - 是否带编
   - 编号
   - 是否物料
   - 是否签字
   - 是否大比例
   - 卡价
6. 卡牌公司是下拉选项。
7. 系列、球员名称、队伍支持基于已有数据的自动补全。

#### 订单管理

1. 保留订单列表布局。
2. 点击订单后弹出订单详情页。
3. 订单支持增删改查。
4. 支持将一个订单绑定到多个藏品。
5. 当绑定成功后，如果藏品的购买时间或购买金额为空，会自动同步订单中的对应信息。
6. 新增订单功能已具备基础版本。

### 2.3 Gallery

1. 支持卡册展示。
2. 支持选择卡册和页码。
3. 支持将藏品放入固定卡位。
4. 一个藏品可以出现在多个展示场景中。

### 2.4 分析

当前已支持：

1. 藏品总数
2. 商品金额
3. 邮费
4. 总金额
5. 售出总额
6. 标签数量
7. 按大类统计

### 2.5 设置

当前已支持：

1. 分类模板编辑
2. 字段管理
3. 全局标签编辑

交互特征：

1. 分类管理、字段管理、全局标签均为可折叠模块。
2. 字段默认折叠展示，只显示字段名称和类型。
3. 点击字段后展开编辑内容。
4. 字段支持拖动调整顺序。
5. 模板变更会立即影响仓库新增藏品表单。

## 3. 当前数据结构

主要对象：

1. `Category`
2. `CustomField`
3. `Tag`
4. `CollectionItem`
5. `Purchase`
6. `SaleRecord`
7. `Album`
8. `AlbumSlot`
9. `DigitalAsset`

当前 `CollectionItem` 重点字段：

1. `categoryId`
2. `imageUrl`
3. `storageLocation`
4. `price`
5. `purchaseDate`
6. `purchaseAmount`
7. `purchaseId`
8. `customValues`

## 4. 本地存储与离线能力

1. 藏品、订单、模板、标签、卡册等数据写入 `localStorage`
2. 页面资源通过 `Service Worker` 做基础缓存
3. 在 `localhost` 开发环境中自动关闭旧的 `Service Worker` 注册，避免缓存旧页面

## 5. 当前已完成事项

1. 产品方向从原生 App 转向 PWA
2. PRD 已同步改为 PWA 优先
3. Expo / React Native 骨架已切换为 Vite Web 项目
4. 仓库、Gallery、分析、设置四个主页面已跑通
5. 仓库按新流程重组
6. 订单管理已具备基础 CRUD
7. 分类模板与字段可编辑
8. 全局标签可编辑
9. 构建与类型检查通过

## 6. 当前限制

1. 图片上传目前使用浏览器本地数据 URL 保存，未接入远端文件存储
2. 拖动排序目前主要实现为字段顺序调整，尚未加入更细的拖拽反馈
3. 订单新增页仍偏基础
4. 藏品详情和订单详情目前采用弹层，不是独立路由页面
5. 在线同步仍处于预留阶段，尚未接入后端

## 7. 本地开发方式

使用 Node 20：

```bash
nvm use 20
```

启动开发环境：

```bash
npm install
npm run dev
```

类型检查：

```bash
npm run typecheck
```

生产构建：

```bash
npm run build
```

## 8. 推荐下一步开发方向

### 8.1 优先级最高

1. 给仓库、藏品详情、订单详情补正式路由，而不只是弹层
2. 接入在线同步方案，先做手动同步，再做自动同步
3. 为图片上传接入真实存储方案

### 8.2 中优先级

1. 完善订单新增和批量导入
2. 给设置页加入模板预设恢复、导出导入
3. 补充字段拖拽反馈和移动端手势体验

### 8.3 后续迭代

1. Gallery 的展板/自由摆放
2. CardHobby CSV 导入页
3. 拍照识别和数字化扫描
4. AI 分析与自然语言筛选
